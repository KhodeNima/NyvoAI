---- NyvoStudio coding style guide ----:

    1. All programmers who program in python, Should follow the PEP8 style guide.

    2. Programmers who use other languages, Should follow their language style guide, But must still adhere to the blank line handling rulesets of PEP8.

    3. All codes must pass the linter CI reviews before merging to the main branch.

    4. All Python codes must be reformatted by the psf/black reformatter before merging to the main branch.

    5. All codes must have the least unit test coverage of ( 70% ) before being merged to the main project branch.

    6. All codes must have the least mutation score of ( %80 ) before being merged to the main project branch.


---- END ----
